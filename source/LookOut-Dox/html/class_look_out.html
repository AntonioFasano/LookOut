Sub LookOut::ExportVcf ( SavePath As  String,
Optional SourceFolder As  String,
Optional DefaultFolder As  Boolean 
)

Export all contact in a folder tree as vCards (.vcf) including Outlook categories.

Parameters
SavePathis the full path of export dir. The tree structure is not kept. It must exist and can have env variables, e.g. %USERPROFILE%\Desktop\Export.
SourceFolderis the Outlook folder to export (subfolders included).
DefaultFolderIf True, SourceFolder (to be omitted) will be the global default contact folder.

The vCard filenames are based on the contact FileAs fields. If a duplicate contact is found, the procedure stops exporting further contacts and skips to the next folder in the queue, if any.

Property Get LookOut::GlobalDefaultPath ( )

Returns the path of the global default contact folder.

Returns
[As String] The path of the global contact folder.

As many commands accept as input the global default contact folder, you may want to learn its path with respect to Outlook (tree) navigation pane. Note: The name of the store contained in Outlook paths is orignally like its related PST file, but the user can change it. With RClick on the store node, check Data File Properties->Advancedto learn about the PST name.

Sub LookOut::MultiVcf ( SourceFolder As  String,

)

Combine single-contact vCards, in a given directory, into a single multi-contact vCard.

Parameters
SourceFolderis the file system folder containing vCards to export (subfolders excluded). It must exist and can have env variables, e.g. USERPROFILE%\Desktop\SourceVcf

Before combining, source vCards are checked to be single-contact vCards. Non vCard files in SourceFolder are ignored.
